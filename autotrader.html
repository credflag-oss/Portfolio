갭상승 스크리닝 전략

전일 종가 대비 시가가 갭상승한 종목을 스크리닝하고, 당일 모멘텀을 활용해 단기 익절을 반복하는 전략입니다.

1
코스피 시장 필터 — 지수 갭 -0.5% 이상 하락 시 당일 전략 스킵
2
갭상승 스크리닝 — 갭상승률 2%~25% 구간, 거래량비율 2배 이상 종목 필터 (동전주·상한가 제외)
3
상위 20종목 선정 — 갭상승률 기준 정렬 후 상위 N개 종목 매수
4
종목당 100만원 투자 — 최대 20종목 동시 보유 (총 투자금 최대 2,000만원)
5
익절 +1% / 손절 -2% — 개별 종목 조건 충족 즉시 청산
6
15:00 강제청산 — 당일 미청산 포지션 장마감 전 전량 청산

주요 설정값 (config.py)

모든 파라미터는 config.py의 CONFIG 딕셔너리에서 한 곳에서 관리합니다.

파라미터 기본값 설명
min_gap_ratio2.0%최소 갭상승률
max_gap_ratio25.0%최대 갭상승률 (상한가 제외)
min_volume_ratio2.0x최소 거래량 배율
top_n20스크리닝 상위 종목 수
invest_per_stock1,000,000원종목당 투자금
max_stocks20최대 동시 보유 종목 수
target_profit+1.0%개별 익절선
stop_loss-2.0%개별 손절선
force_close_time15:00장마감 강제청산 시각
fee_rate0.015%편도 수수료
tax_rate0.18%거래세
market_gap_filter-0.5%코스피 갭 하락 필터 (이하 스킵)

프로젝트 구조

main.py

진입점. 전체 백테스트 / 빠른 테스트(5거래일) 모드 선택

config.py

모든 파라미터 중앙 관리 (갭비율, 투자금, 익절/손절, 기간 등)

engine/backtest.py

일별 시뮬레이션 코어. 스크리닝 → 매수 → 청산 로직 순차 실행

engine/position.py

포트폴리오 매니저. 보유 종목, 현금 잔고, 손익 집계

screener/volatility.py

갭상승률·거래량비율 기반 종목 스크리닝 로직

data/fetcher.py

KIS API 연동. 거래일 목록·분봉·일봉 데이터 수신

kis/

KIS Open API 인증 및 REST 요청 래퍼

utils/report.py

콘솔 요약 출력 및 backtest_result.csv 저장
