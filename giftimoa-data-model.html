Firestore 컬렉션 경로 구조
// ── 최상위 컬렉션 ────────────────────────────────────── users/ {userId} ← 사용자 프로필 kakaoId: string familyId: string? fcmToken: string notifSettings: { d7: bool, d3: bool, d1: bool } families/ {familyId} ← 가족 그룹 memberIds: string[] inviteCode: string ← 6자리, 48h 유효 gifticons/ {gifticonId} ← 기프티콘 본문 ownerId: string familyId: string? barcodeNumber: string ← AES-256 암호화 expiryDate: timestamp status: GifticonStatus └─ subcollection usageLogs/ {logId} ← 이벤트 이력 notifications/ {notifId} ← 유저별 알림 피드 userId: string ← 인덱스 필드 type: NotifType isRead: boolean // ── Firebase Storage 경로 ────────────────────────────── gifticons/ {userId}/ {gifticonId}/ original.jpg ← 원본 이미지 thumb_300.jpg ← 썸네일 (Cloud Function 자동 생성)
5
Firestore 복합 인덱스 전략
컬렉션 인덱스 필드 정렬 방향 사용 쿼리
gifticons ownerId + status + expiryDate ASC / ASC / ASC 내 기프티콘 목록 (만료일 오름차순)
gifticons familyId + isShared + status + expiryDate ASC / ASC / ASC / ASC 가족 공유 기프티콘 필터링
gifticons status + expiryDate ASC / ASC 만료 임박 배치 처리 (Cloud Scheduler)
notifications userId + isRead + sentAt ASC / ASC / DESC 안 읽은 알림 최신순 조회
notifications userId + sentAt ASC / DESC 알림 전체 목록 최신순 조회
families inviteCode — 초대 코드로 가족 조회
users kakaoId — 카카오 로그인 시 사용자 조회
6
Firebase 보안 규칙 핵심 원칙
🔒
본인 데이터만 쓰기 가능
gifticons 쓰기 시 request.auth.uid == resource.data.ownerId 검증. 타인의 기프티콘은 읽기만 가능 (가족 공유 범위 내).
👨‍👩‍👧
가족 멤버만 공유 기프티콘 조회
isShared == true인 기프티콘은 families/{familyId}.memberIds에 포함된 userId만 읽기 허용. 외부 사용자 접근 차단.
👑
ADMIN만 구성원 삭제 가능
families 문서의 memberIds 배열 수정은 adminId == request.auth.uid인 경우만 허용. CHILD 계정은 gifticons 등록 불가.
🔑
바코드 번호 암호화
barcodeNumber 필드는 클라이언트에서 AES-256으로 암호화 후 저장. 복호화 키는 Android Keystore에 보관. Firestore 규칙으로 원본 평문 저장 차단 불가 → 클라이언트 강제 암호화 구현 필요.
📋
usageLogs 는 append-only
서브컬렉션 usageLogs는 create만 허용, update/delete 금지. 이력 위변조 방지 목적.
7
오프라인 지원 & 동시성 처리
📴 오프라인 바코드 표시
Firestore의 persistenceEnabled = true 설정으로 로컬 캐시 활성화. 네트워크 없이도 기프티콘 상세 및 바코드 조회 가능. 단, 사용 완료 처리는 온라인 시 동기화.
⚡ 동시 사용 충돌 방지
Firestore Transaction으로 status 변경 처리. 두 가족 구성원이 동시에 "사용 완료"를 누르면 먼저 커밋된 트랜잭션만 성공 — 나머지는 실패 후 UI 재조회.
🔔 만료 알림 배치
Cloud Scheduler가 매일 오전 9시 실행 → expiryDate 기준 D-7, D-3, D-1 해당 기프티콘 조회 → notifSentDays로 중복 발송 방지 → FCM으로 푸시 전송.
🖼️ 이미지 최적화
업로드 시 Cloud Functions가 트리거되어 thumb_300.jpg 자동 생성. 목록 화면은 썸네일만 로드, 상세 화면에서만 원본 로드 → 데이터 절약 & 속도 개선.
8
카카오 로그인 → Firebase 연동 플로우
// ① 카카오 로그인 (Android Kakao SDK) KakaoSDK.login() → OAuthToken(accessToken, refreshToken) ↓ // ② 카카오 사용자 정보 조회 UserApiClient.me() → KakaoUser { id, nickname, profileImage } ↓ // ③ 서버(Cloud Functions)에 카카오 토큰 전달 POST /auth/kakao { kakaoAccessToken } ↓ // ④ Cloud Functions: 카카오 토큰 검증 → Firebase Custom Token 발급 admin.auth().createCustomToken(kakaoId, { nickname, profileImage }) → firebaseCustomToken ↓ // ⑤ 클라이언트: Firebase Custom Token으로 Firebase 로그인 FirebaseAuth.signInWithCustomToken(firebaseCustomToken) → FirebaseUser { uid } ← 이 uid 가 userId ↓ // ⑥ users/{uid} 문서 upsert (최초 가입 또는 토큰 갱신) firestore.collection('users').doc(uid).set({ kakaoId, nickname, profileImage, fcmToken, updatedAt }, { merge: true })
