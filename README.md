# 소비 관리 앱 기획 요약

## 개요
- 사용자는 구매한 상품 정보를 등록하고, 등록된 내역을 확인할 수 있습니다.
- 결제 내역(영수증) 이미지를 업로드하면, API가 이미지에서 **상품명, 카테고리, 가격, 날짜**를 추출하여 자동 등록합니다.
- 이후 등록된 내역에 **평점**을 매겨 과소비를 관리할 수 있는 앱을 목표로 합니다.

## 핵심 사용자 흐름
1. **영수증 업로드** → 이미지에서 구매 정보를 자동 추출
2. **자동 등록 결과 확인** → 필요 시 수동 수정
3. **구매 내역 목록/상세 조회**
4. **평점 입력** → 소비 만족도/과소비 관리

## MVP 기능 범위
- 영수증 이미지 업로드
- OCR 기반 정보 추출 (상품명, 카테고리, 가격, 날짜)
- 구매 내역 CRUD
- 구매 내역 목록/상세 조회
- 구매 내역 평점 입력/수정

## 확장 기능 아이디어
- 카테고리별/기간별 소비 분석
- 과소비 경고/알림
- 목표 예산 설정
- 월별 리포트

## 데이터 모델 초안
- **Purchase**
  - id
  - item_name
  - category
  - price
  - purchased_at
  - rating
  - receipt_image_url
  - created_at
  - updated_at

## API 초안
- `POST /receipts`
  - request: 이미지 파일
  - response: 추출된 구매 내역 후보 리스트
- `POST /purchases`
  - request: 구매 내역 정보
  - response: 생성된 구매 내역
- `GET /purchases`
  - response: 구매 내역 목록
- `GET /purchases/{id}`
  - response: 구매 내역 상세
- `PATCH /purchases/{id}`
  - request: 구매 내역/평점 수정

## 개발 시작
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

## 화면/UX 초안
- 홈: 해당 월 소비 요약을 **파이형 그래프**로 표시하고, 아래로 스크롤하면 날짜순 세부 내역 표시
- 빈 상태: 내역이 없을 때 "신나는 소비를 해볼까요?" 문구와 귀여운 돈 모양 표시, 클릭 시 **결제내역 신규추가**로 이동
- 상단 우측: **결제내역 신규추가** 바로가기
- 업로드: 영수증 이미지 업로드
- 확인/수정: 추출 결과 검토 및 수정
- 내역: 리스트/필터/정렬
- 상세: 내역 상세, 평점 입력
